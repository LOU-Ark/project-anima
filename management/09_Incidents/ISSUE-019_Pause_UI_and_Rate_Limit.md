# ISSUE-019: PAUSEボタンUI不備とAPIレートリミット超過 (EXTERNAL)

## 1. 事象
- **報告日時**: 2026-04-26
- **指摘内容**: 
    1. PAUSEボタンのデザインが全体の世界観（ダーク・ネオン）と合わず「ださい」。
    2. 21歳以降、ノートが一切生成されなくなった（APIの回数制限の疑い）。

## 2. 根本原因分析 (RCA)
1. **デザインの欠落**: `index.html` にPAUSEボタンを追加した際、`style.css` 側で専用のクラス定義を行わず、ブラウザのデフォルトスタイル（白背景に黒文字）がそのまま露出していた。
2. **APIレートリミット（429エラー）**: 
    - Gemini APIの無料枠は「15リクエスト/分」。
    - シミュレーターをスピードx10で回すと、1.5秒（実時間）に1回のペースで「日常観測」の判定が走り、数十秒で制限に到達する。
    - `.env` には自動ローテーション用の複数のキー（5つ）が用意されていたが、`ai-client.js` 側で `keys[0]` に固定参照する実装になっており、負荷分散が機能していなかった。

## 3. 知的財産への変換 (Asset Conversion)
- **修正内容**:
    - `style.css`: PAUSEボタンに対してグラスモーフィズムとネオンボーダー（`#00f2fe`等）を適用し、ホバー時のグリッチ・アニメーション等を追加してサイバー感を演出する。
    - `ai-client.js`: API呼び出し時にエラー（429）を検知した場合、即座に次のAPIキーへローテーション（ラウンドロビン）して再試行する自己修復ロジックを実装する。

## 4. 再発防止策
- `VERIFICATION_STANDARD.md` の「視覚層」に、「全UIコンポーネントがテーマ（Aurora Dark）に準拠していること」を明記。
- 「動的テスト」に、「API依存機能はレートリミット時の挙動（ローテーションやリトライ）を確認すること」を追加。
