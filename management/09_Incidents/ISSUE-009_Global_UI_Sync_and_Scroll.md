# ISSUE-009: スクロールバグの根本原因とグローバルUI同期の欠如

**報告日**: 2026-04-25
**分類**: EXTERNAL (UI/UX / Architecture)
**ステータス**: 修正中

---

## 1. 指摘内容
1. ログ画面にスクロールバー（スライダ）が表示されず、依然として過去のログが読めない。
2. テーマスライダーや一時停止ボタンがゴースト画面にしかなく不便。ポータルやWebアプリ側からも制御したい。
3. テーマをライトモードにした際の文字色や背景色の工夫が足りず見づらい。

## 2. 根本原因分析 (RCA)
- **スクロールバーの消失**: Flex/Gridレイアウトにおける典型的な罠。親コンテナに `overflow: hidden`、子コンテナに `min-height: 0` と `height: 100%` を設定していなかったため、コンテナがコンテンツに合わせて無限に拡張してしまっていた。
- **アーキテクチャの局所化**: Dual Product（複数アプリ連携）を標榜していながら、状態管理（Pause状態やテーマ）を `index.html` 内のローカル変数に閉じ込めてしまっていた。
- **カラーパレット設計の甘さ**: 背景色（0〜100）の変化に対し、テキスト色を単なる二値（白か黒か）で切り替えただけだったため、グラスモーフィズム（半透明パネル）とのコントラストが崩壊していた。

## 3. 再発防止策 (仕組みによる解決)
- [ ] CSSで `overflow-y: auto` を使う際は、必ず親コンテナの高さ制限（`flex: 1; min-height: 0;` など）をセットで検証することを `VERIFICATION_STANDARD.md` に追加する。
- [ ] マルチウィンドウ構成のアプリ群において、全体に影響を与える環境変数（テーマ、シミュレーション状態）は必ず `localStorage` をハブとした Publish/Subscribe モデルで全画面同期させる設計をチーム標準とする。
