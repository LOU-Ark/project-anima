# ISSUE-007: 非同期API呼び出しによる時系列の崩壊（Race Condition）

**報告日**: 2026-04-25
**分類**: EXTERNAL (バグ修正)
**ステータス**: 修正中

---

## 1. 指摘内容
Ghost Simulator を高速再生すると、Webアプリ側に表示されるメモの順序が「80歳 → 22歳 → 45歳」のように時系列がめちゃくちゃになる。

## 2. 根本原因分析 (RCA)
- **技術的原因**: `setInterval` による時間経過は同期的（一定ペース）に進むが、Gemini API を呼び出す `injectNote` 関数は非同期処理（ネットワーク遅延に依存）であった。
- APIへのリクエストは年齢順に発行されるが、レスポンスが返ってくる順序は保証されない。レスポンスを受け取った順に `localStorage` の先頭（`unshift`）に追加していたため、順序が逆転した。
- **検証の不備**: 高速再生（x60等）というエッジケースにおける非同期処理の競合（Race Condition）テストが漏れていた。

## 3. 再発防止策 (仕組みによる解決)
- [ ] ゴーストシミュレーター内にイベントキュー（FIFO）を実装し、前の推論が完全に終わってストレージに書き込まれるまで、次の推論を開始しない「直列処理」を保証する。
- [ ] 非同期処理をループ内で連続発火させるアーキテクチャでは、必ずキューイングまたはミューテックス制御を行うことを `VERIFICATION_STANDARD.md` 等の開発定石として扱う。
