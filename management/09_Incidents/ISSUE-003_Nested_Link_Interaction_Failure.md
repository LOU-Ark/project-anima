# ISSUE-003: ポータル内の履歴リンク・クリック不能障害

**報告日**: 2026-04-25
**分類**: EXTERNAL (ユーザー指摘)
**ステータス**: 修正中

---

## 1. 指摘内容
ポータル内の Milestone 03 カードにおいて、履歴リスト（01, 02, 03）が表示されているものの、それらをクリックしても報告書が開かない。リンクとしての機能が死んでいる。

## 2. 根本原因分析 (RCA)
- **物理的原因**: 
    1. カードを `a` タグから `div` タグへ変更した際、CSSのセレクタ（`.card`）が `a` 前提の挙動（インラインブロック要素等）を失い、レイアウトが不安定になった。
    2. カード内のリンク要素（`a`）に対して、親要素のスタイルや重なり順が干渉し、ポインタイベントを遮断していた可能性がある。
- **背景要因**: 
    「UIの見た目」の変更に注力し、HTML構造の変更に伴う「インタラクションの再検証（疎通確認）」をエージェントが怠った。

## 3. 再発防止策 (仕組みによる解決)
- [ ] リンク要素には必ず `position: relative; z-index: 10;` を明示し、イベントの捕捉を確実にする。
- [ ] 修正後は、**必ず `browser_subagent` を用いて、各個別リンクを実際にクリックし、遷移先のレンダリングを確認する手順**を検証基準に追加する。

---
> [!CAUTION]
> 「見えるのに動かないUI」は、ユーザーの信頼を最も損なう。動的な検証こそがエージェントの生命線である。
