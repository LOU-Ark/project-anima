# ISSUE-012: ゴーストシミュレーター停止問題 (EXTERNAL)

## 1. 事象
- **報告日時**: 2026-04-26 10:17
- **指摘内容**: 修正後もシミュレーター（A-E EMOTION）が動かない（年齢・感情レベルが更新されない）。
- **ステータス**: 分析中

## 2. 根本原因分析 (RCA)
### 現状の仮説
- **仮説A (Syntax Error)**: 直近のコード修正（マルチパッチ適用時）に構文エラーが混入し、JavaScriptの実行自体が停止している。
- **仮説B (Async Hang)**: `loadApiKeys` 等の非同期処理が解決（resolve）されず、後続の `initializeAI` やタイマー開始をブロックしている。
- **仮説C (Logic Error)**: `isGenerating` や `globalPaused` の初期値判定に矛盾があり、タイマーが即座に `return` されている。

### なぜ内部検証で見逃したか
- 修正後にブラウザでの実機確認を手動のみで行い、自動化されたE2Eテスト（Playwright等）による動作保証を実施していなかったため。
- 「コードが正しいはず」という思い込みにより、APIキーがない環境でのフォールバック動作の網羅的な検証を怠った。

## 3. 知的財産への変換 (Asset Conversion)
- **修正**: E2Eテストによる動作確認が取れるまで修正を継続する。
- **再発防止策**: 
    - `Rules/VERIFICATION_STANDARD.md` を更新。UIロジックの修正後は必ず「タイマーが進行すること」を確認する自動テストを必須とする。
    - `Skills/AUTO_TEST/SKILL.md` に、非同期初期化を伴うシミュレーター系のテスト項目を追加。

## 4. 対応計画
1.  **実機調査**: ブラウザサブエージェントを使用してコンソールログを詳細に取得。
2.  **修正**: エラー箇所の特定と修正。
3.  **検証**: E2Eテスト（スクリプト）による自動検証。
4.  **報告**: 修正内容と再発防止策の提示。
