# ISSUE-018: AIクライアントのサイレント実行不全 (EXTERNAL)

## 1. 事象
- **報告日時**: 2026-04-26
- **指摘内容**: 「まったくメモしなくなりました」
- **発生状況**: ISSUE-017（完全自律型トピック生成への移行）の適用直後、シミュレーションが進行してもゴースト画面に一切のノート（ログ）が出力されなくなった。

## 2. 根本原因分析 (RCA)
1. **SDKパッケージ名の混同**: `GoogleGenerativeAI` クラスを使用する際、旧パッケージ `@google/generative-ai` ではなく、新パッケージ `@google/genai` を誤ってインポートしていた。これによりJavaScript実行時にモジュール解決エラーが発生し、`AIEngine` 自体が未ロード状態となった。
2. **フォールバックの欠如による不可視化**: ISSUE-017で「固定シナリオの代替表示」を削除したため、AIエンジンが未ロードの場合、エラーがコンソールにのみ出力され、画面上には何も表示されない「サイレントな実行不全」に陥った。
3. **脆弱なJSON抽出**: GeminiのレスポンスにMarkdownバッククォートが含まれた場合、単純な `replace` ではJSONパースエラーを引き起こす懸念があった。

## 3. 知的財産への変換 (Asset Conversion)
- **修正の実施**:
    - `ai-client.js` のインポートURLを `https://esm.sh/@google/generative-ai` へ修正。
    - `JSON.parse` の前処理として、正規表現 `match(/\{[\s\S]*\}/)` を用いて確実にJSONブロックのみを抽出するロバストな処理を実装。

## 4. 再発防止策
- このインシデントは、変更作業をドキュメント化せずに直接修正したことで、ポータルへの同期が漏れる原因となった。
- 今後、すべての外部フィードバック（エラー報告）は、**必ず `EXTERNAL_FEEDBACK_FLOW` を通してISSUE化し、ポータルに同期させる**手順をワークフロー自体に組み込んだ。
