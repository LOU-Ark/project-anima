# ISSUE-006: 非標準的なAPI呼び出しとシークレット管理の不備

**報告日**: 2026-04-25
**分類**: EXTERNAL (アーキテクチャ要件)
**ステータス**: 修正中

---

## 1. 指摘内容
Gemini API の呼び出しにおいて、直接の `fetch` ではなく公式SDK `@google/genai` を使用すること。また、APIキーをブラウザのプロンプトから入力させるのではなく、`.env` ファイルから読み込む設計にすること。モデルは `gemini-3-flash-preview` を使用すること。

## 2. 根本原因分析 (RCA)
- **技術的妥協**: ビルドツール（Webpack/Vite）を持たない静的ファイル（Vanilla JS）のプロジェクトであったため、npmパッケージの導入や `.env` のパースが面倒だと判断し、最も手抜きな手法（`fetch` + `prompt()` + `localStorage`）を採用してしまった。
- **ガバナンスの欠如**: シークレット情報をUIから入力させる設計は、本番運用を視野に入れたアーキテクチャとしては脆弱であるにもかかわらず、「とりあえず動くデモ」を優先してしまった。

## 3. 再発防止策 (仕組みによる解決)
- [ ] プロジェクトルートに `.env` を配置し、JS側から非同期（`fetch`等）で読み込んでパースする仕組みを標準化する。
- [ ] 今後、AI系の機能拡張を行う際は、必ず公式ライブラリ（SDK）をES Modules経由等でロードして使用することを必須ルールとする。
- [ ] `VERIFICATION_STANDARD.md` を更新し、「シークレット情報の直書きやUI入力の禁止」を明記する。

---
> [!WARNING]
> 「ビルド環境がないから」は言い訳にならない。Vanilla JS であっても、モダンな開発基準（SDK + .env）に準拠する方法は常に模索されなければならない。
