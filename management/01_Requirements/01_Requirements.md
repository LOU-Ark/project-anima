# 01_要件定義書: Aurora Notes (Google Keep-style)

## 1. プロジェクト概要
Google Keepの「直感的な操作性」と「カード型UI」をベースに、Aurora Governance基準の「プレミアムな美的品質」を融合させた、個人向け次世代タスク管理・メモアプリの開発。

## 2. ターゲット
- **一般生活者**: 複雑な設定を嫌い、直感的に思考を整理したいユーザー。
- **デザイン重視のユーザー**: ツールに美しさと洗練さを求めるユーザー。

## 3. 機能要件
- [ ] **メイソンリー・レイアウト**: メモの長さに応じて柔軟に配置が変わるグリッドシステム。
- [ ] **メモ作成・編集**: タイトルと本文を入力可能。
- [ ] **カラー・ラベリング**: Auroraパレット（発光色を含む）に基づいたメモの背景色設定。
- [ ] **検索機能**: タイトルや本文からリアルタイムに絞り込み。
- [ ] **アーカイブ/ゴミ箱**: 整理と削除のライフサイクル管理。
- [ ] **ローカル保存**: ブラウザの `localStorage` を使用した永続化。

## 4. 非機能要件
- [ ] **文字コード**: UTF-8 (BOMなし)
- [ ] **パフォーマンス**: 100件以上のメモでもスムーズなスクロールとアニメーション。
- [ ] **レスポンシブ**: スマートフォンおよびデスクトップの両方で最適な表示。
- [ ] **アクセシビリティ**: 視認性の高いコントラストと、Aurora基準のタイポグラフィ。

## 5. 制約事項
- [ ] 外部DBは使用せず、フロントエンド完結型とする。
- [ ] バニラJSおよびCSSを基本とし、軽量かつ高速な動作を保証する。

## 6. 技術スペック (Technical Specifications)
- **Frontend**: HTML5, CSS3 (Custom Properties), JavaScript (ES6+)
- **Storage**: Browser LocalStorage
- **Icons**: Lucide-like SVG Icons

## 7. 成功の定義 (Definition of Success)
- ユーザーが1秒以内にメモを開始できる直感的なUIが実現されていること。
- Auroraのビジュアル基準（Glassmorphism, Neon Accents）が適用され、プレミアムな操作体験を提供していること。
- メモの作成、編集、削除、検索が不具合なく動作すること。

## 8. デザイン方針
- **Aurora Minimalist**: 余計な装飾を削ぎ落としつつ、グラスモーフィズムによる奥行きと、グラデーションによる「驚き」を与える。
- **Typography**: Inter (Sans-serif) を使用し、モダンな印象を与える。
- **Color Palette**: ダークモードを標準とし、メモの色は Aurora Neon パレットを採用。
