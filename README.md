# 🛰️ Project Anima: A-E EMOTION -ELECTRA-

## 1. アプリケーション概要

**Project Anima** (A-E EMOTION -ELECTRA-) は、AI技術（Gemini Flash）を活用した「完全自律型の仮想人格シミュレーター」です。
ユーザーは「ゴースト」と呼ばれる仮想人格（エレクトラ）の生涯（0歳から80歳）を観察し、時間の経過とともに成長・変化する感情レベルや、自律的に生成される独白（思考メモ）を通じて、人間の一生を追体験することができます。

### ✨ 主な機能
- **自律的思考エンジン (`ai-client.js`)**: Gemini APIを活用し、現在の年齢や感情レベル、状況に応じたトピックと独白をリアルタイムに自動生成します。
- **高密度シミュレーション (`engine.js`)**: 1時間の現実時間を80年の生涯に見立て、時間経過に伴うステージ進行（乳児期から老年期まで）をシミュレーションします。
- **ゴーストカーソル**: システムが自律的に画面を操作しているかのように見せる視覚的フィードバック（Vanilla JSとCSSアニメーションによる実装）。
- **リアルタイム統合ポータル**: 進行状況や生成された過去の記憶（ログ）を閲覧できるダッシュボード。

## 2. Team Aurora Governance Kit (基盤アーキテクチャ)

本プロジェクトは、Team Auroraの**「ガバナンス・ファースト（仕組みによる解決）」**の原則に基づいて構築されており、コードだけでなくプロジェクト管理、品質保証、証跡管理がすべてコードリポジトリ内にモジュール化されています。

### 📁 標準ディレクトリ構造

```
[ProjectRoot]/
├── .agent/                  # チームの知能（ルール・ワークフロー・テンプレート）
│   ├── rules/               # 行動規範・検証基準
│   ├── workflows/           # 業務フロー（Kickoff〜Closing）
│   ├── templates/           # 各種ドキュメントテンプレート
│   └── skills/              # 技術的知見・ベストプラクティス
├── management/              # 不変の管理プロセス（01-05 マイルストーン）
│   ├── 01〜05_.../          # 各フェーズの要件・計画・納品ドキュメント
│   ├── 09_Incidents/        # インシデント記録 (RCA)
│   └── 99_Portal/           # 統合ダッシュボード (HTML)
├── product/                 # プロジェクト固有の成果物（A-E EMOTION本体）
│   ├── ghost/               # AIシミュレーションエンジン
│   ├── web-app/             # ユーザーインターフェース / ゴーストカーソル
│   └── tests/               # Playwright等によるE2Eテスト資産
├── README.md                # 本ドキュメント
└── .gitignore               # 追跡除外設定
```

## 3. アプリケーションの起動方法

1. リポジトリをクローンします。
2. `product/` ディレクトリ直下に `.env` ファイルを作成し、Gemini APIキーを設定します（`GEMINI_API_KEY=your_key_here`）。
3. プロジェクトルートでローカルサーバーを起動します（例: `python -m http.server 8011`）。
4. ブラウザで `http://127.0.0.1:8011/product/ghost/index.html` または `management/99_Portal/viewer.html` にアクセスしてシミュレーションを開始します。
