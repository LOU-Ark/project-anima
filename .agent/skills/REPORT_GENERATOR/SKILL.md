# 検証報告書自動生成スキル (REPORT_GENERATOR)

## 1. 目的
作業の実施と並行して検証報告書を生成し、作業内容の透明性と再現性を確保する。

## 2. 実行手順
1. **初期化**: 作業開始時に `management/09_Verification/` 内に `[ID]_Verification_[TaskName].md` を作成する。
2. **並行記録**: 作業の各ステップ（ファイル作成、コマンド実行等）が終わるたびに、その結果（エビデンス）を報告書に追記する。
   - **重要**: 外部エビデンス（画像・動画）を使用する際は、必ず `management/09_Verification/assets/` に複製してから相対パスでリンクすること。
   - **必須**: 永続化や状態遷移を伴うタスクでは、必ずブラウザサブエージェントの録画ファイルを動画エビデンスとして添付すること。
3. **最終検証**: 作業終了時に `VERIFICATION_STANDARD.md` に基づく三層検証を行い、最終判定を下す。
4. **同期**: 報告書が完成したことを確認してから、WBSを `done` に更新する。

## 3. 推奨アクション
- コマンド実行時の出力（成功/失敗）をそのままコピーしてエビデンスとする。
- ポータルの表示確認結果を必ず一言添える。
