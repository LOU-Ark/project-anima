# SKILL: ISSUE_MANAGER

## 1. 概要
このスキルは、プロジェクト中に発生した不具合や要件の齟齬を検知し、`ENG-002` (イシュー管理ライフサイクル) に基づいて解決までを自動化・構造化して管理するためのものです。

## 2. 発動トリガー
- ユーザーからの「不具合」「文字化け」「表示がおかしい」等の指摘を受けた時。
- セルフチェック（ブラウザサブエージェント等）で期待値との乖離を発見した時。
- 新機能の実装中に、既存機能への影響（退行）を検知した時。

## 3. 実行フロー (ENG-002 準拠)
1. **[ERROR] 記録**: `management/03_Issues/` に新しいイシューファイルを作成。
2. **[CONFIRMATION] 調査**: ログ・ブラウザコンソール・コードを調査し、原因をエビデンス付きで記録。
3. **[PLAN] 計画**: 修正計画を策定。この際、`templates/` に既存の解決策がないか確認する。
4. **[EXECUTION] 実装**: 計画に基づき修正。必要に応じて `templates/` を更新または新規作成する。
5. **[VERIFICATION] 検証**: `VERIFICATION_STANDARD.md` に基づき検証。スクリーンショット等をエビデンスとして保存。
6. **[REPORT] 報告**: ユーザーへ解決を報告し、イシューをクローズ。

## 4. 知的財産の保護
- 解決策が汎用的なものである場合（例：ハイパーガントのCSS等）、必ず `.agent/templates/` にテンプレートとして抽出し、チームの資産として蓄積すること。
