# 標準開発ワークフロー (DEVELOPMENT_FLOW)

## 1. ワークフロー概要
タスクの着手から完了までの標準的なプロセスを定義します。

## 2. フェーズとステップ

### Phase 1: 準備 (Plan)
- [ ] 依頼内容の理解と、必要なリソースの確認。
- [ ] WBSを `active` に更新。

### Phase 2: 実施と同時報告 (Do & Document)
- [ ] **REPORT_GENERATORスキルの起動**: `management/09_Verification/` に報告書を新規作成。
- [ ] 実作業の実施（コード作成、コマンド実行等）。
- [ ] 各ステップの結果を報告書にリアルタイムで追記。

### Phase 3: 検証 (Check)
- [ ] `VERIFICATION_STANDARD.md` に基づく三層検証。
- [ ] 報告書に最終判定を記載。

### Phase 4: 完了 (Act)
- [ ] 報告書の保存。
- [ ] WBSを `done` に更新。
