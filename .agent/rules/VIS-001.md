# VIS-001: プレミアムUI/UX視覚基準

## 1. デザイン原則
Aurora Frameworkに基づくアプリケーションおよびドキュメントは、以下の原則に従うこと。
- **Glassmorphism (グラスモーフィズム)**: 背景を透過・ぼかすことで、奥行きと透明感を表現する。
- **Hyper-Visuals (ハイパー・ビジュアル)**: ネオンカラー、グラデーション、発光エフェクトを用い、ユーザーに「驚き」を与える視覚体験を提供する。

## 2. ガントチャート基準 (Hyper Gantt)
プロジェクト計画を可視化するガントチャートは、以下の仕様を「標準」とする。
- **テーマ**: ダークテーマ (`#0a0c10` 前後)。
- **タスクバー**: `linear-gradient(90deg, #00f2fe 0%, #7367f0 100%)` 等の鮮やかなグラデーション。
- **エフェクト**: タスクバーに外光（`box-shadow` によるグロー効果）を付与する。
- **レイアウト**: WBS項目とタイムラインが明瞭に分離されたグリッドレイアウト。

## 3. ドキュメント閲覧基準
すべてのMarkdownドキュメントは、以下の要素を含む「洗練されたビューア」を通じて提供されなければならない。
- **フォント**: `Inter` (欧文) および `Noto Sans JP` (和文) を標準とし、システムフォントに頼らない。
- **コード表示**: `highlight.js` によるシンタックスハイライト。
- **Mermaid統合**: Mermaid記法を自動検知し、上記「Hyper Gantt」基準でレンダリングする。

## 4. 検証プロセス
- 実装されたUIが本基準に準拠しているか、ブラウザサブエージェントによるスクリーンショット検証を必須とする。
